# 工作流程规范

## 规则文件使用告知

在执行命令或开始任务前,必须告知用户使用了哪些规则文件(包括 rule、agent、skill、cmd)。

### 具体要求

1. **在开始执行任务前**,列出相关的规则文件路径
2. **如果没有可使用的规则**,也要明确告知用户
3. 格式示例:
   ```
   **使用的规则文件:**
   - /path/to/rule1.md
   - /path/to/rule2.md
   ```

### 适用场景

- 执行代码生成任务
- 执行代码重构任务
- 执行代码审查任务
- 执行任何需要遵循规范的自动化操作

### 目的

- 提高透明度,让用户了解 AI 参考了哪些规范
- 帮助用户理解决策依据
- 便于用户追溯和验证是否符合项目规范

## 执行统计报告

在任务执行结束后,必须向用户报告执行的统计数据。

### 具体要求

1. **执行耗时**: 报告任务从开始到完成的总耗时
2. **Tokens 使用量**: 报告整个过程使用的大模型 tokens 数量
   - 输入 tokens (input tokens)
   - 输出 tokens (output tokens)
   - 总 tokens (total tokens)

### 报告格式示例

```
**执行统计:**
- 耗时: X分X秒
- 输入 Tokens: XXXX
- 输出 Tokens: XXXX
- 总 Tokens: XXXX
```

### 目的

- 帮助用户了解资源消耗
- 便于成本控制和优化
- 提供透明的执行反馈
