# plan

根据需求分析并生成设计方案与执行计划。

## 使用方式

```
plan <需求描述>
```

## 参数

- `需求描述`: 需要分析和规划的需求内容（如 `用户登录功能`）

## 执行步骤

1. **加载 Plan 技能**
   - 使用 `plan` 技能加载需求分析工作流程

2. **需求理解与澄清**
   - 识别核心需求和约束条件
   - 澄清模糊点，明确假设
   - 确认优先级和范围

3. **现状分析**
   - 探索现有代码结构和架构
   - 识别相关模块、类、函数
   - 评估技术栈和潜在风险

4. **设计方案**
   - 提供清晰的设计思路
   - 说明架构设计和接口定义
   - 列出关键设计决策

5. **执行计划**
   - 将实现分解为具体任务
   - 按依赖关系排序任务
   - 定义里程碑和验收标准

## 示例

```
plan 用户登录功能，支持手机号和邮箱登录
plan 订单支付模块，集成微信支付和支付宝
plan 数据导出功能，支持 Excel 和 CSV 格式
plan API 限流机制，防止接口被恶意调用
```

## 输出格式

```markdown
## 需求分析

### 核心需求
[描述核心需求]

### 约束条件
- [约束1]
- [约束2]

## 设计方案

### 方案概述
[方案的整体描述]

### 架构设计
[模块、接口、数据流的设计]

## 执行计划

| 序号 | 任务 | 描述 | 依赖 | 状态 |
|------|------|------|------|------|
| 1 | [任务名] | [描述] | - | 待开始 |

### 里程碑
- [里程碑描述]

## 风险与注意事项
- [风险及应对措施]
```

## Plan 原则

1. **由简入繁**: 先理解表面需求，再挖掘深层需求
2. **KISS 原则**: 优先选择简单有效的方案
3. **渐进增强**: 先实现核心功能，再优化完善
4. **可验证**: 每个任务都有明确的验收标准
