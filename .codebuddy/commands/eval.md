# 评估命令

管理评估驱动的开发工作流。

## 用法

`/eval [define|check|report|list] [功能名称]`

## 定义评估

`/eval define 功能名称`

创建新的评估定义：

1. 创建 `.claude/evals/功能名称.md` 使用模板：

```markdown
## 评估：功能名称
创建时间：$(date)

### 能力评估
- [ ] [能力描述1]
- [ ] [能力描述2]

### 回归评估
- [ ] [现有行为1仍然正常]
- [ ] [现有行为2仍然正常]

### 成功标准
- 能力评估 pass@3 > 90%
- 回归评估 pass^3 = 100%
```

2. 提示用户填写具体标准

## 检查评估

`/eval check 功能名称`

运行功能评估：

1. 从 `.claude/evals/功能名称.md` 读取评估定义
2. 对每个能力评估：
   - 尝试验证标准
   - 记录通过/失败
   - 在 `.claude/evals/功能名称.log` 记录尝试
3. 对每个回归评估：
   - 运行相关测试
   - 与基线比较
   - 记录通过/失败
4. 报告当前状态：

```
评估检查：功能名称
========================
能力：X/Y 通过
回归：X/Y 通过
状态：进行中 / 就绪
```

## 报告评估

`/eval report 功能名称`

生成全面的评估报告：

```
评估报告：功能名称
=========================
生成时间：$(date)

能力评估
----------------
[评估-1]：通过 (pass@1)
[评估-2]：通过 (pass@2) - 需要重试
[评估-3]：失败 - 见备注

回归评估
----------------
[测试-1]：通过
[测试-2]：通过
[测试-3]：通过

指标
-------
能力 pass@1：67%
能力 pass@3：100%
回归 pass^3：100%

备注
-----
[任何问题、边缘情况或观察]

建议
--------------
[可发布 / 需要工作 / 阻塞]
```

## 列出评估

`/eval list`

显示所有评估定义：

```
评估定义
================
feature-auth      [3/5 通过] 进行中
feature-search    [5/5 通过] 就绪
feature-export    [0/4 通过] 未开始
```

## 参数

$ARGUMENTS:
- `define <名称>` - 创建新评估定义
- `check <名称>` - 运行并检查评估
- `report <名称>` - 生成完整报告
- `list` - 显示所有评估
- `clean` - 删除旧评估日志（保留最近10次运行）
