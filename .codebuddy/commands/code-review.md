# 代码审查

对未提交变更进行全面的安全和质量审查：

1. 获取变更文件：git diff --name-only HEAD

2. 对每个变更文件，检查：

**安全问题（严重）：**
- 硬编码的凭证、API密钥、令牌
- SQL注入漏洞
- XSS漏洞
- 缺少输入验证
- 不安全的依赖
- 路径遍历风险

**代码质量（高）：**
- 函数超过50行
- 文件超过800行
- 嵌套深度超过4层
- 缺少错误处理
- console.log语句
- TODO/FIXME注释
- 公共API缺少JSDoc

**最佳实践（中）：**
- 变更模式（应使用不可变模式）
- 代码/注释中使用表情符号
- 新代码缺少测试
- 无障碍问题（a11y）

3. 生成报告：
   - 严重程度：严重、高、中、低
   - 文件位置和行号
   - 问题描述
   - 建议修复

4. 如果发现严重或高问题则阻止提交

绝不批准有安全漏洞的代码！
